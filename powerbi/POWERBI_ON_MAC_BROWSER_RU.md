# Power BI на Mac без Windows — пошаговый путь

## 1. Что понадобится

- актуальный Safari, Chrome, Edge или Firefox;
- рабочая или учебная учётная запись Microsoft;
- доступ к `app.powerbi.com` или `app.fabric.microsoft.com`;
- папка `Halyk-BI-interview/data` с шестью исходными CSV.

Личные адреса Gmail и Hotmail напрямую для Power BI Service не поддерживаются. Если рабочей учётной записи нет, Microsoft предлагает создать тестовый Microsoft 365 tenant с адресом `onmicrosoft.com`.

## 2. Проверка лицензии

1. Откройте `https://app.powerbi.com`.
2. Войдите рабочей или учебной учётной записью.
3. Нажмите на аватар справа сверху.
4. Проверьте тип лицензии.

Free позволяет создавать личный контент в `My workspace`. Для совместной работы и отправки отчёта коллегам обычно нужен Pro, PPU или Trial. Для демонстрации через показ экрана достаточно личного workspace.

Если доступны возможности Fabric, можно активировать 60-дневный Fabric Trial через Account manager → Start trial.

## 3. Создание семантической модели

Названия пунктов могут немного отличаться из-за обновлений интерфейса.

1. Откройте `My workspace`.
2. Выберите `New item` или `Create`.
3. Выберите `Semantic model` / `Get data`.
4. Выберите `Text/CSV` или загрузку локального CSV.
5. Первым загрузите `clients.csv`.
6. Выберите создание семантической модели, а не автоматический финальный отчёт.
7. Откройте созданную модель через `Open data model`.
8. Через `Get data` добавьте остальные таблицы:
   - `accounts.csv`;
   - `trades.csv`;
   - `cash_operations.csv`;
   - `instruments.csv`;
   - `managers.csv`.

Если веб-интерфейс предлагает загрузить только один файл за раз, это нормально: добавляйте таблицы последовательно в одну открытую модель.

## 4. Проверка типов

Через `Transform data` откройте Power Query Online.

- идентификаторы — Whole number / Int64;
- даты — Date;
- `trade_amount`, `commission`, `amount` — Decimal number или Fixed decimal/Currency;
- статусы, тикеры и каналы — Text.

Проверьте, что первая строка CSV использована как заголовки.

## 5. Создание связей

Создайте связи по файлу `MODEL_RELATIONSHIPS.md`. Направление фильтрации — от справочника к таблице событий. Не включайте двунаправленную фильтрацию без необходимости.

После загрузки создайте таблицу `Календарь` из начала файла `DAX_MEASURES.dax`, затем добавьте связи с датами сделок и денежных операций.

## 6. Добавление DAX

В редакторе модели выберите `New measure` и по очереди добавьте меры из `DAX_MEASURES.dax`.

Сначала добавьте:

1. Торговый оборот.
2. Комиссионный доход.
3. Количество сделок.
4. Активные клиенты.
5. Пополнения, выводы и чистый приток.
6. Конверсию в первую сделку.
7. Неактивных клиентов 90 дней.
8. Меры качества.

После каждой группы добавляйте простую карточку или таблицу и сверяйте результат с `mart_brokerage_monthly.csv`.

## 7. Создание отчёта

1. Нажмите `New report` / `Create report`.
2. Выберите пустой отчёт.
3. Соберите страницы по `REPORT_PAGES_RU.md`.
4. Используйте `VISUALIZATION_THEORY_RU.md` как шпаргалку выбора диаграмм.
5. Сохраните отчёт в `My workspace`.

## 8. Что получится

Отчёт будет храниться в облаке и открываться на Mac в браузере. Его можно редактировать без Power BI Desktop. При бесплатной лицензии его можно использовать для личной демонстрации и показа экрана; возможность отправить коллегам интерактивную ссылку зависит от лицензии и capacity workspace.

## 9. Ограничения веб-версии

- Desktop имеет более полный набор подключения, моделирования и внешних инструментов.
- Не все Desktop-функции и пользовательские визуалы доступны онлайн.
- Локальный CSV не обновится автоматически после изменения файла: потребуется повторная загрузка либо OneDrive/SharePoint/Fabric-источник.
- Внешний вид на Mac может немного отличаться из-за шрифтов.
- HTML-дашборд нельзя конвертировать одной кнопкой; визуалы пересобираются на той же семантической модели.

## 10. MCP

Modeling MCP не нужен, чтобы собрать первую версию. Он автоматизирует семантическую модель, но не создаёт страницы визуализации. К нему имеет смысл возвращаться после того, как рабочая модель уже создана в Power BI Service/Fabric или сохранена как PBIP.

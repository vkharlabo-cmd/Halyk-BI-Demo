# Словарь демонстрационных данных

Все данные синтетические и сгенерированы скриптом `scripts/generate_demo_data.py`. Они не относятся к Halyk или реальным клиентам.

## clients.csv — клиенты, 320 строк

| Поле | Значение |
|---|---|
| client_id | Уникальный идентификатор клиента |
| registration_date | Дата регистрации |
| client_segment | Mass, Affluent, Premium или Corporate |
| manager_id | Ответственный менеджер |
| acquisition_channel | Канал привлечения |
| status | Active, Dormant или Closed |

## accounts.csv — брокерские счета, 320 строк

`account_id`, `client_id`, дата открытия, тип и статус счёта. Связь с клиентом: `clients.client_id = accounts.client_id`.

## trades.csv — сделки, 1 851 строка

Одна строка = одна сделка. Поля: идентификатор сделки, клиент, дата, инструмент, сумма оборота, комиссия, BUY/SELL. Связи с `clients` и `instruments` выполняются по идентификаторам.

## cash_operations.csv — денежные операции, 1 130 строк

Одна строка = одно пополнение или вывод. Поля: операция, счёт, клиент, дата, тип операции, сумма.

## instruments.csv — инструменты, 12 строк

Справочник тикеров, типов инструментов и рынков: акции, облигации, ETF и валютные пары.

## managers.csv — менеджеры, 8 строк

Справочник менеджеров и подразделений Retail, Premium, Digital и Corporate.

## mart_brokerage_monthly.csv — месячная витрина, 18 строк

Одна строка = один календарный месяц. Содержит активных и новых клиентов, сделки, оборот, комиссию, пополнения, выводы, чистый приток и изменение комиссии к предыдущему месяцу.

## data_quality_checks.csv — контроль качества, 5 строк

Намеренно созданные ошибки: дубль сделки, отсутствующий клиент, отсутствующий инструмент, отрицательная комиссия и денежная операция без счёта/клиента.

# Инструкция для пользователя — Claude Code

> Этот файл объясняет как работает папка проекта, созданная из шаблона,
> что за что отвечает, и какие команды давать Claude Code.

---

## Структура папки проекта

```
my-project/
├── CLAUDE.md                  ← Протокол работы (загружается автоматически)
├── CONTEXT.md                 ← Описание проекта: стек, цель, деплой
├── PLAN.md                    ← Фазы и задачи проекта
├── DECISIONS.md               ← Архитектурные решения (ADR)
├── CONTRACT_DRAFTING.md       ← Правила подготовки договоров и партнерских соглашений
├── SKILLS.md                  ← Справочник всех скиллов
├── CODE_MAP.md                ← Карта кода: файлы, endpoints, компоненты
├── .claude/
│   └── settings.json          ← Настройки Claude Code для этого проекта
├── memory-templates/          ← Шаблоны памяти (копируются при онбординге)
├── task_plan.md               ← Активный план задач (создаётся скиллом)
├── progress.md                ← Лог сессий + БЫСТРОЕ ВОЗОБНОВЛЕНИЕ
└── findings.md                ← Исследования и заметки текущей сессии
```

---

## Как это работает

**CLAUDE.md** — загружается автоматически при открытии папки в VS Code. Содержит все правила, протоколы и красные линии. Читать вручную не нужно — Claude уже видит его.

**CONTEXT.md** — заполняется при первом запуске (онбординге). Хранит описание проекта, стек, URL деплоя. Claude использует его для понимания контекста.

**progress.md** — после каждой задачи Claude записывает сюда «БЫСТРОЕ ВОЗОБНОВЛЕНИЕ» — 5 строк с текущим состоянием. При следующей сессии достаточно написать «продолжай».

**CODE_MAP.md** — карта кода. Claude обновляет её при добавлении новых endpoint'ов, компонентов, страниц. Вместо поиска по коду — читать нужный раздел отсюда.

**CONTRACT_DRAFTING.md** — чеклист для договоров: referral, White Label,
подрядчики, комиссии, международные партнеры, санкции, privacy, применимое
право и спорные условия.

**Память** (`~/.claude/projects/[путь]/memory/`) — создаётся при онбординге. Хранит профиль, историю сессий, правила. Загружается автоматически, не трогать вручную.

## Первый запуск (онбординг)

Если в папке есть файл `_SETUP_NEEDED` — открой папку в VS Code и Claude сам начнёт онбординг. Он задаст 5 вопросов:

1. Название проекта
2. Что строим (1-2 предложения)
3. Стек (frontend / backend / БД)
4. Деплой (Vercel / VPS / локально)
5. GitHub репозиторий (ссылка или «нет»)

После ответов Claude заполнит все файлы и настроит память. Больше ничего делать не нужно.

## Как правильно давать задачи

| Что хочешь | Как написать |
|-----------|-------------|
| Новая фича | «Добавь [описание]. Жду план и подтверждения.» |
| Баг | «Сломалось [что]. Симптом: [описание]. Найди и почини.» |
| Быстрая правка | «Поменяй [что] на [что] в [файл]» |
| Статус задач | «статус» или «что сейчас» |
| Продолжить работу | «продолжай» — Claude прочитает БЫСТРОЕ ВОЗОБНОВЛЕНИЕ |
| Деплой | «задеплой» или `/vercel:deploy` |
| Ревью кода | `/review` |

Claude всегда:
- Пишет список задач **до** первого действия
- Показывает план для сложных задач и ждёт подтверждения
- Делает git commit после каждой завершённой задачи
- Тестирует результат на реальном URL (не только в редакторе)

## Скиллы — команды `/`

| Скилл | Что делает | Когда |
|-------|-----------|-------|
| `/planning-with-files:plan` | Создаёт структурированный план | Сложная задача (Claude вызывает сам) |
| `/planning-with-files:status` | Показывает текущий план | Проверить прогресс |
| `/test` | Smoke-тесты системы | После любого изменения кода |
| `/simplify` | Ревью качества кода | Изменено 3+ файла или 100+ строк |
| `/security-review` | Аудит безопасности | Изменил авторизацию / API / данные |
| `/review` | Ревью pull request | Перед мержем |
| `/ui-ux-pro-max:ui-ux-pro-max` | UI/UX дизайн | Новый компонент / редизайн |
| `/vercel:deploy` | Деплой на Vercel | Готово к выпуску |
| `/vercel:logs` | Логи деплоя | Что-то сломалось после деплоя |
| `/claude-api` | Anthropic SDK | Работа с AI API |
| `/loop` | Повторять по интервалу | Мониторинг, polling |
| `/schedule` | Запуск по расписанию | Cron-задачи |
| `/update-config` | Настройки Claude Code | Изменить хуки / permissions |
| `/humanizer` | Убрать AI-стиль | Маркетинговые тексты |

## Что НЕ нужно делать вручную

- Читать CLAUDE.md — Claude видит его автоматически
- Создавать task_plan.md — это делает `/planning-with-files:plan`
- Обновлять progress.md — Claude делает это после каждой задачи
- Обновлять CODE_MAP.md при добавлении endpoint'а — Claude сам обновит

---

## Если что-то пошло не так

| Проблема | Решение |
|---------|---------|
| Claude не следует протоколу | «Выполни Block 0.1 из CLAUDE.md» |
| Потерян контекст после сводки | «Прочитай БЫСТРОЕ ВОЗОБНОВЛЕНИЕ из progress.md» |
| Скилл не запустился | «Запусти claude skills» — Claude сообщит об ошибке |
| Нужно откатить изменения | `git log` / `git revert` в папке проекта |
| Непонятно что сейчас делается | `/planning-with-files:status` |

# Как выбирать визуализацию в Power BI

Главное правило: сначала определите вопрос, затем выбирайте график. Не начинайте с желания использовать красивый визуал.

## Одно ключевое значение — карточка

Используйте для оборота, комиссии, активных клиентов, конверсии и количества ошибок.

Карточка отвечает: «Какое значение сейчас?» Она не объясняет причину и динамику, поэтому рядом нужен график или детализация.

## Динамика во времени — линейный график

Используйте для оборота, комиссии, активных клиентов, пополнений и выводов по месяцам.

Линия лучше столбцов, когда важен тренд. Не объединяйте показатели с сильно различающимися масштабами на одной оси.

## Сравнение категорий — горизонтальные столбцы

Используйте для инструментов, менеджеров, каналов и сегментов.

Горизонтальная ориентация удобна для длинных названий. Сортируйте по убыванию и показывайте Top N, если категорий много.

## Состав целого — накопленные столбцы

Используйте, чтобы показать структуру оборота по типам инструментов или клиентским сегментам во времени.

100% накопленные столбцы показывают доли, обычные накопленные — абсолютные суммы. Круговая диаграмма допустима только для 2–5 устойчивых категорий; она плохо подходит для точного сравнения.

## Переход между этапами — воронка

Используйте для цепочки: регистрация → открытие счёта → первая сделка → активный клиент.

Воронка корректна только тогда, когда каждый следующий этап является подмножеством предыдущего.

## Изменение от компонентов — водопад

Используйте для пополнений, выводов и чистого притока либо для объяснения изменения комиссионного дохода.

Водопад показывает, какие компоненты увеличили или уменьшили итог.

## Два числовых показателя — точечная диаграмма

Используйте для поиска связи: активные клиенты менеджера ↔ комиссия или оборот ↔ средняя комиссия.

Она помогает увидеть выбросы и сегменты, но не доказывает причинность.

## Точные значения и детализация — таблица или матрица

Используйте, когда нужно сверять конкретные цифры, раскрывать иерархию или выгружать данные.

Матрица подходит для «подразделение → менеджер → клиент». Не превращайте весь отчёт в таблицы: это инструмент анализа, а не только выгрузка.

## Распределение — гистограмма

Используйте для распределения размера сделок, комиссии на клиента или количества сделок на клиента. Помогает увидеть типичный диапазон и выбросы.

## География — карта

Используйте только при наличии реального географического вопроса. Не добавляйте карту ради украшения.

## Контроль качества — таблица, карточки и условное форматирование

Для ошибок важнее точность и действие, чем декоративный график. Показывайте тип ошибки, количество, критичность, владельца и статус исправления.

## Частые ошибки начинающего

- слишком много цветов;
- две несопоставимые меры на одной оси;
- круговая диаграмма с десятью категориями;
- отсутствие единиц измерения;
- графики без управленческого вопроса;
- использование суммы там, где нужна доля или среднее;
- сравнение неполного месяца с полным без отметки MTD;
- отсутствие даты обновления и определения метрики;
- визуализация сырых данных до проверки качества.

## Шпаргалка

| Вопрос | Визуал |
|---|---|
| Какое значение сейчас? | Карточка |
| Как меняется во времени? | Линия |
| Кто больше или меньше? | Горизонтальные столбцы |
| Из чего состоит итог? | Накопленные столбцы |
| Где теряются клиенты? | Воронка |
| Что сформировало изменение? | Водопад |
| Есть ли связь между показателями? | Точечная диаграмма |
| Нужны точные строки и детализация? | Таблица или матрица |
| Как распределены значения? | Гистограмма |
| Есть ли ошибки данных? | Карточки + таблица + условное форматирование |

# Halyk BI — демонстрационный проект кандидата

Практический проект из чата «Собеседование в Халык»: прототип управленческой BI-витрины для брокерской компании.

## Открыть демо

1. Запустить в корне проекта: `python3 -m http.server 8765`.
2. Открыть: `http://127.0.0.1:8765/dashboard/`.

Дашборд содержит шесть страниц: «Обзор для руководства», «Активность клиентов», «Доход и инструменты», «Качество данных», «Данные и SQL», «Как пользоваться».

## Что показать на собеседовании

- [dashboard/index.html](dashboard/index.html) — интерактивный прототип дашборда.
- [CASE_STUDY.md](CASE_STUDY.md) — готовый рассказ на 3–5 минут.
- [DEMO_GUIDE.md](DEMO_GUIDE.md) — пошаговая инструкция по проверке и демонстрации.
- [FULL_PIPELINE_GUIDE.md](FULL_PIPELINE_GUIDE.md) — полный путь от генерации данных и SQL до выводов руководства.
- [powerbi/README_RU.md](powerbi/README_RU.md) — комплект переноса в Power BI: Power Query, DAX, связи, тема и макеты страниц.
- [sql/brokerage_mart.sql](sql/brokerage_mart.sql) — PostgreSQL: модель, месячная витрина, конверсия, неактивные клиенты, проверки качества и индексы.
- [data/](data/) — шесть исходных таблиц, месячная витрина и контроль качества.
- [source/chat_full.md](source/chat_full.md) — полный сохранённый контекст исходного диалога.

## Модель данных

- Справочники: клиенты, счета, инструменты, менеджеры и календарь.
- События: сделки и денежные операции.
- Уровень детализации витрины: один календарный месяц.
- Намеренно встроены пять ошибок качества для демонстрации контрольного контура.

## Перегенерация

`python3 scripts/generate_demo_data.py`

Генерация детерминирована seed-значением, поэтому метрики воспроизводимы.
